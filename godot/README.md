# HexCar 3D (Godot)

3D-порт игры HexCar (см. корневой `../README.md`) на **Godot 4.7**, вид сверху.
Вся игровая логика перенесена 1:1 из `../game.html`; рендер — настоящий 3D.

## Запуск

Из корня репозитория:

```bash
./Godot_v4.7-stable_win64.exe --path godot
```

Или открыть папку `godot/` в редакторе Godot 4.7 и нажать ▶ (главная сцена — `Main.tscn`).

## Сборка standalone .exe

Нужны шаблоны экспорта 4.7 (один раз: в редакторе *Проект → Экспорт → Управление шаблонами → Скачать*).
Затем из корня репозитория:

```bash
./Godot_v4.7-stable_win64_console.exe --headless --path godot \
  --export-release "Windows Desktop" build/HexCar3D.exe
```

Получается один самодостаточный файл `godot/build/HexCar3D.exe` (контент встроен).
Заставка Godot отключена (`boot_splash/show_image=false` в `project.godot`).

## Что внутри

- **`Main.gd`** — весь проект в одном скрипте: геометрия плиток Труше, три дороги-дуги
  на плитку, движение машинок по дугам, переходы между плитками, тороидальные порталы,
  вращение ±60°, три режима, очки, а также процедурная сборка 3D-сцены.
- **`Main.tscn`** — корневая сцена (`Node3D` со скриптом).

## 3D-особенности

- Дороги — настоящие 3D-ленты; на пересечении внутри плитки верхняя дорога физически
  приподнимается — получается **эстакада** (машинка проезжает под мостом). На стыках
  плиток высота = 0, поэтому сеть стыкуется без разрывов.
- Плитки вращаются плавной анимацией; декор (деревья, кусты, цветы, камни) крутится вместе с плиткой.
- Направленный свет с тенями, процедурное небо, наклонная камера сверху.
- **Оформление света** (меню справа): пресеты *Ночь / Сумерки / День*, сохранение/загрузка
  своих настроек (`user://look.cfg`).
- **Погода** — переключаемая в меню (кнопка «Погода: Нет/Дождь/Снег»): настоящие
  3D-частицы (`GPUParticles3D`) над всем полем, падают на карту. Дождь — косые
  вытянутые струи; снег — мягкие хлопья, медленно кружат (турбулентность).
  Видна перспектива (ближе крупнее, дальше мельче). Важно: у частиц задан
  широкий `visibility_aabb` на весь объём падения — иначе эмиттер высоко над
  кадром отсекался бы вместе со всеми каплями.
- **Стиль рисунка** — полноэкранный пост-шейдер: *Нет · Тун · Акварель · Гуашь ·
  Карандаш · Карандаш 2 · Контур+цвет · Тун градиент · Контур+цвет+тун*.
  Карандаш/Карандаш 2 — контур по Canny с антиалиасингом (разная ширина
  диапазона порогов → меньше/больше деталей, чистый фон); Контур+цвет — те же
  линии **multiply** на цветной рендер; Тун градиент — ч/б мягкие ступени
  яркости; Контур+цвет+тун — контур + цвет + ч/б тун-градиент (multiply)
  вместе, плюс **толстый тун-контур** (широкорадиусный Sobel `thick_edge`,
  жирные силуэты деревьев и дорог поверх тонких AA-линий). Выбор сохраняется
  вместе с настройками света.

## Управление

Наведи курсор на плитку и поверни её на ±60°:
- стрелки `←` / `→`
- клавиши `A` / `S`
- колесо мыши
- зажми ЛКМ и тяни вбок

## Режимы

**Кольца** · **Погоня** · **Бандит** — как в оригинале (выбор в меню).

## Бот (режим Кольца)

Кнопка слева сверху или клавиша `B` запускает бота, который сам крутит плитки,
собирая **самую большую петлю**. Алгоритм — хилл-климбинг по ориентациям плиток:
поле раскладывается на петли (полудуга → сосед — это перестановка, её циклы — петли),
максимизируется `Σ (плиток в петле)²` (выпуклая цель → одна огромная петля). При
застревании — случайные пертурбации, лучшая найденная конфигурация запоминается и
восстанавливается в конце. Из случайного поля 8×5 собирает петлю ~35–36 из 40 плиток.
Пока бот работает, спавн машинок и аварии приостановлены.
