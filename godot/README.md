# HexCar 3D (Godot)

3D-порт игры HexCar (см. корневой `../README.md`) на **Godot 4.7**, вид сверху.
Вся игровая логика перенесена 1:1 из `../game.html`; рендер — настоящий 3D.

## Запуск

Из корня репозитория:

```bash
./Godot_v4.7-stable_win64.exe --path godot
```

Или открыть папку `godot/` в редакторе Godot 4.7 и нажать ▶ (главная сцена — `Main.tscn`).

## Сборка standalone .exe

Нужны шаблоны экспорта 4.7 (один раз: в редакторе *Проект → Экспорт → Управление шаблонами → Скачать*).
Затем из корня репозитория:

```bash
./Godot_v4.7-stable_win64_console.exe --headless --path godot \
  --export-release "Windows Desktop" build/HexCar3D.exe
```

Получается один самодостаточный файл `godot/build/HexCar3D.exe` (контент встроен).
Заставка Godot отключена (`boot_splash/show_image=false` в `project.godot`).

## Что внутри

- **`Main.gd`** — весь проект в одном скрипте: геометрия плиток Труше, три дороги-дуги
  на плитку, движение машинок по дугам, переходы между плитками, тороидальные порталы,
  вращение ±60°, три режима, очки, а также процедурная сборка 3D-сцены.
- **`Main.tscn`** — корневая сцена (`Node3D` со скриптом).

## 3D-особенности

- Дороги — настоящие 3D-ленты; на пересечении внутри плитки верхняя дорога физически
  приподнимается — получается **эстакада** (машинка проезжает под мостом). На стыках
  плиток высота = 0, поэтому сеть стыкуется без разрывов.
- Плитки вращаются плавной анимацией; декор (деревья, кусты, цветы, камни) крутится вместе с плиткой.
- Направленный свет с тенями, процедурное небо. **Камера по умолчанию** — вид
  сверху на длинном фокусе (узкий FOV ≈16° → почти ортогонально, минимум
  перспективных искажений), кадрируется так, чтобы было видно всё поле (с учётом
  правой панели). Пресет можно вернуть кнопкой **Камера → «Вид сверху»** в меню
  (`_apply_cam_top()`); орбита/зум/наклон по-прежнему доступны мышью и Q/E/R/F/Z/X.
- **Оформление** (меню справа) — две независимые оси, которые комбинируются:
  - **Время суток**: *Ночь / Сумерки / День* — базовый свет, небо, солнце/луна.
  - **Сезон**: *Лето / Осень / Зима / Весна* — вид поля + сезонная
    цветокоррекция света поверх времени суток (множители в `SEASON_GRADE`).
  Можно сочетать любые пары (Зима + Ночь, Осень + День и т. д.).
  Сохранение/загрузка своих настроек (`user://look.cfg`), сезон тоже сохраняется.
- **Сезоны поля** меняют землю и весь декор:
  - *Лето* — зелёная трава, цветы, зелёные деревья.
  - *Осень* — увядающая жёлто-оливковая трава с опавшими листьями, листва
    деревьев жёлто-оранжево-красная, меньше цветов.
  - *Зима* — снег вместо земли, без травы/цветов, деревья «в снегу»
    (подмороженная листва + белые шапки), кусты и камни → сугробы;
    автоматически включается снегопад.
  - *Весна* — свежая ярко-зелёная трава, цветущие деревья (бело-розовые
    гроздья), много цветов.
  Реализовано так: раскладка декора (позиции травы и объектов) генерируется
  один раз и хранится на плитке (`t.items`, `t.grass_x`); декор каждого сезона
  живёт в отдельном контейнере `t.deco[сезон]`, строится лениво при первом
  показе и переиспользует ту же раскладку — поэтому смена сезона **не сбрасывает
  состояние головоломки** (повороты плиток сохраняются).
- **Погода** — переключаемая в меню (кнопка «Погода: Нет/Дождь/Снег»): настоящие
  3D-частицы (`GPUParticles3D`) над всем полем, падают на карту. Дождь — частые
  тонкие косые струи (густой); снег — мелкие хлопья, рождаются **по всей высоте
  столба** (от земли до ~520) при умеренной гравитации, поэтому видны равномерно
  и у земли, и наверху, а не тонким слоем у точки спавна, и при этом заметно
  стремятся вниз. Видна перспектива (ближе крупнее, дальше мельче). Края эффекта
  не обрываются квадратом: материал частиц — шейдер (`WEATHER_SHADER`), который
  гасит прозрачность к краям поля по **эллиптической** маске (нормировка на
  полуразмер поля), поэтому на виде сверху дождь/снег круглo растворяются вокруг
  поля. Важно: у частиц задан широкий `visibility_aabb` на весь
  объём падения — иначе эмиттер высоко над кадром отсекался бы вместе со всеми
  каплями.
- **Края поля** не выглядят прямоугольным «островом» статичной земли: земля —
  шейдер (`GROUND_SHADER`) с **эллиптической** маской (смещение от центра поля
  нормируется на полуразмер поля, изолинии — эллипсы, поэтому на виде сверху
  переход круглый, без углов/квадрата). Земля плотная только над зоной плиток
  (`GROUND_FADE_START`), а дальше широко уходит в прозрачность к
  `GROUND_FADE_END`; плоскость заметно больше поля (`_ground_plane_size()`),
  чтобы затухание ушло в 0 ещё **до** её прямоугольного края — самой кромки не
  видно. Плюс **цвет фона у горизонта подгоняется под цвет освещённой земли**
  сезона (`_ground_base_color()` × свет пресета в `_apply_look()`), но
  делается **темнее и насыщеннее** самой земли (`s * 1.35`, `darkened(0.48)`),
  поэтому переход поля в фон плавный, а сам фон мягко обрамляет поле и не спорит
  с ним. Маска погоды (`WEATHER_SHADER`) — такая же эллиптическая, дождь/снег
  круглo растворяются вокруг поля.
- **Стиль рисунка** — полноэкранный пост-шейдер: *Нет · Тун · Акварель · Гуашь ·
  Карандаш · Карандаш 2 · Контур+цвет · Тун градиент · Контур+цвет+тун*.
  Карандаш/Карандаш 2 — контур по Canny с антиалиасингом (разная ширина
  диапазона порогов → меньше/больше деталей, чистый фон); Контур+цвет — те же
  линии **multiply** на цветной рендер; Тун градиент — ч/б мягкие ступени
  яркости; Контур+цвет+тун — контур + цвет + ч/б тун-градиент (multiply)
  вместе, плюс **толстый тун-контур** (широкорадиусный Sobel `thick_edge`,
  жирные силуэты деревьев и дорог поверх тонких AA-линий). Выбор сохраняется
  вместе с настройками света.

## Управление

Наведи курсор на плитку и поверни её на ±60°:
- стрелки `←` / `→`
- клавиши `A` / `S`
- колесо мыши
- зажми ЛКМ и тяни вбок

## Режимы

**Кольца** · **Погоня** · **Бандит** — как в оригинале (выбор в меню).

## Бот (режим Кольца)

Кнопка слева сверху или клавиша `B` запускает бота, который сам крутит плитки,
собирая **самую большую петлю**. Алгоритм — хилл-климбинг по ориентациям плиток:
поле раскладывается на петли (полудуга → сосед — это перестановка, её циклы — петли),
максимизируется `Σ (плиток в петле)²` (выпуклая цель → одна огромная петля). При
застревании — случайные пертурбации, лучшая найденная конфигурация запоминается и
восстанавливается в конце. Из случайного поля 8×5 собирает петлю ~35–36 из 40 плиток.
Пока бот работает, спавн машинок и аварии приостановлены.
